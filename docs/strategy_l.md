# Strategy L: 300成长纯MA60趋势择时

> 极简趋势跟踪 -- 收盘价在MA60上方满仓，下方空仓

## 1. 策略概述

**核心思想**: 用最简单的趋势跟踪方法——60日均线——对沪深300成长全收益指数做择时。收盘价在MA60上方就满仓，下方就空仓。没有任何额外的滤波、确认、止损等复杂机制。

| 维度 | 内容 |
|------|------|
| 策略类型 | 趋势跟踪 / 择时 |
| 标的 | 沪深300成长全收益指数 (H00918) |
| 对应ETF | 310398 (沪深300成长ETF) |
| 信号频率 | 日频 (每个交易日检查) |
| 交易成本 | 单边8bp |
| 数据源 | 中证指数公司 (CSIndex API) |

## 2. 策略逻辑

```
每个交易日收盘后:

  1. 计算当日收盘价和60日简单移动平均 (MA60)
  2. 判断:
     - 收盘价 > MA60 -> 目标仓位 = 100% (满仓)
     - 收盘价 <= MA60 -> 目标仓位 = 0% (空仓)
  3. 信号延迟: 今天生成的信号，明天执行 (T日信号, T+1执行)
  4. 若信号与昨天不同 -> 调仓 (买入或卖出)
```

### 核心参数

| 参数 | 值 | 说明 |
|------|---|------|
| trend_ma | 60 | 趋势判断均线 (约3个月) |
| txn_cost_bps | 8 | 单边交易成本 |

**为什么选MA60?** MA60大约是3个月的均线，能够过滤掉大的下跌趋势（如2008金融危机、2015杠杆牛崩盘、2018贸易战、2022大熊市），同时不会过于迟钝。这是在research/trend_reversal_combo.py中对243种参数组合全因子回测后筛选出的最优配置。

## 3. 回测表现

**回测期**: 2005 ~ 2026 (约20.5年)

| 指标 | MA60策略 | 买入持有 |
|------|---------|---------|
| CAGR | 18.9% | 14.5% |
| Sharpe | 1.050 | - |
| Max Drawdown | -31.7% | >-70% |
| Calmar | ~0.60 | ~0.21 |

### 择时价值分析

- CAGR提升: +4.4个百分点 (18.9% vs 14.5%)
- MDD大幅改善: -31.7% vs >-70%
- 20年累计收益差异巨大 (复利效应)

### 关键历史表现

- **2008年**: 大部分时间空仓，躲过-65%暴跌
- **2015年**: 6月前持仓享受牛市收益，7月后及时退出
- **2018年**: 大部分时间空仓，躲过-30%下跌
- **2022年**: 大部分时间空仓，躲过成长股大幅回撤
- **震荡市**: 来回交易产生摩擦成本（这是趋势策略的固有代价）

## 4. 标的选择

**为什么选300成长而非沪深300?**

沪深300成长指数是沪深300中成长风格的子集，弹性更大：
- 上涨时涨幅更大（成长风格溢价）
- 下跌时跌幅也更大（但被趋势过滤器截断）
- 趋势跟踪在高弹性标的上效果更好
- 全收益指数包含分红再投资

## 5. 偏差处理

| 偏差类型 | 处理方式 |
|---------|---------|
| 前瞻偏差 | 信号T日生成，T+1执行 (`shift(1)`) |
| 交易成本 | 含单边8bp |
| 选择偏差 | 低 (标准化宽基指数) |
| 数据源可信度 | 高 (中证指数官网API，权威数据) |

## 6. 适用场景

- **资金定位**: 大盘成长择时配置
- **适合投资者**: 能接受-30%回撤、追求长期高收益的投资者
- **互补策略**: 与策略D/G（个股低波）、策略A（债券）形成风格互补
- **操作难度**: 极低 (只需买卖一只ETF)

## 7. 局限性

1. MDD -31.7%仍然较大（对比买入持有已大幅改善）
2. 震荡市频繁交易产生摩擦成本
3. 趋势信号有滞后性，无法在最高点卖出或最低点买入
4. 依赖成长风格持续有效，若市场风格永久转向价值，策略可能长期空仓
5. 单一ETF策略，无分散化

## 8. 研究来源

策略L来源于`research/trend_reversal_combo.py`的大规模参数搜索：
- 搜索范围: 243种趋势+反转组合
- 搜索维度: 均线类型(SMA/EMA) x 均线周期(20/40/60/120/250) x 指数标的 x 是否叠加反转信号
- 最终选择: 纯MA60 + 300成长指数 (最高Sharpe，最简逻辑)
